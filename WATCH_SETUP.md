# SoundCue Watch — 설치·사용·개인화·리포트 가이드

> **Galaxy Watch 7 Ultra** Standalone 앱.
> Flip 6 과 연결되면 **양방향 청각 브리지**로 동작.
> 아기 → 부모 (소리 알림) / 부모 → 아기 (수화 → TTS 음성)

---

## 0. 양방향 흐름

```
[아기 → 부모]
폰 마이크/버튼 → Gemma 4 해석 → 워치 화면·강진동·알림
  "아기가 울고 있어요 (배고픔 가능성)"

[부모 → 아기]
워치 퀵 버튼 or 제스처 → (워치 TTS 즉시 + 폰 Gemma 맥락 문장)
  워치 스피커: "아가 배고프니? 맘마 줄까?"
  폰 스피커:  "배고팠구나, 아빠가 금방 우유 타올게."  ← Gemma가 상황·시간·페르소나 반영
```

---

## 1. 사용자 프로필 설정

> 첫 실행 시 or 설정 화면에서 입력. Gemma 가 문장 생성할 때 이 맥락을 반영.

### 필수 설정

| 항목 | 예시 | Gemma 반영 |
|---|---|---|
| **아기 이름** | 하은이 | "하은아, 배고프니?" (이름 호명) |
| **아기 생년월일 / 개월 수** | 2025-10-15 (6개월) | 개월 수 기반 리포트 비교 기준 |
| **부모 호칭** | 엄마 / 아빠 | "아빠가 여깄어" vs "엄마가 여깄어" |
| **말투 스타일** | 다정한 / 차분한 / 밝은 | 톤 조절 (아래 참고) |

### 선택 설정

| 항목 | 예시 | 효과 |
|---|---|---|
| 주요 언어 | 한국어 / English | TTS + Gemma 출력 언어 |
| 야간 모드 시간대 | 22:00 ~ 06:00 | 야간엔 톤·볼륨 자동 조절 |
| 수유 간격 기본값 | 3시간 | "마지막 수유로부터 X시간" 계산 |

## 2. 시간대별 톤 자동 조절

> 같은 제스처라도 **시간에 따라 Gemma가 생성하는 문장의 톤이 달라짐**.

### 예시: `hungry` 제스처

| 시간 | Gemma 생성 문장 | 톤 |
|---|---|---|
| **낮 2시** | "하은아, 배고프구나! 아빠가 맘마 줄게~" | 밝고 활기찬 |
| **밤 11시** | "쉿, 배고팠구나… 아빠가 조용히 우유 타올게." | 낮고 부드러운 속삭임 |
| **새벽 3시** | "괜찮아, 아빠 여기 있어. 우유 금방 줄게." | 최소한의 짧은 문장 |

### 구현 원리

Gemma 프롬프트에 `time_of_day` + `night_mode` 컨텍스트 주입:

```
현재 시각: 23:12 (야간 모드)
→ 톤 지시: 속삭이듯, 짧게, 자극 최소화
→ TTS pitch: -0.3, speed: 0.85
```

## 3. 페르소나 반영 상세

### 입력 조합 → Gemma 출력

```
Input:
  event = baby_cry (배고픔 패턴)
  gesture = hungry
  persona = 아빠
  baby_name = 하은이
  time = 23:12 (야간)
  style = 다정한

Gemma Output:
  "하은아, 아빠가 금방 우유 타올게. 조금만 기다려줄래?"
  TTS pitch: low, speed: slow
```

```
Input:
  event = baby_laugh
  gesture = well_done
  persona = 엄마
  baby_name = 하은이
  time = 14:30 (낮)
  style = 밝은

Gemma Output:
  "하은아, 잘했어! 우리 하은이 진짜 대단하다~"
  TTS pitch: normal, speed: normal
```

### Safety Context — 지속 울음 시 가이드

아기가 **연속 3회 이상 울음 이벤트**가 감지되고 부모가 `hungry` 제스처만 반복하면:

```
Gemma가 추가 워치 알림 생성:
"하은이가 계속 울고 있어요.
 혹시 어디 아픈 건 아닌지 확인해 보세요.
 체온이나 기저귀 상태도 봐주세요."
```

→ 단순 "배고파요" 반복이 아니라 **상황 에스컬레이션** (건강 확인 유도).

## 4. 알림 → 수화 발화 워크플로우

### 실제 사용 시나리오

```
1. [폰 감지] 아기가 울기 시작
2. [워치 알림] 강진동 + "아기가 울어요 (배고픔 가능)" 카드
3. [부모 행동] 알림 내용 확인 → 아기 근처로 이동
4. [워치 퀵 버튼] 🍼 배고프니? 탭
5. [워치 스피커] 즉시 "하은아, 배고프니?" 재생
6. [폰 스피커] Gemma가 맥락 반영 → "배고팠구나, 아빠가 금방 우유 타올게."
7. [부모] 아기 수유 시작
8. [이벤트 로그] "울음 → 수유 응답" 타임라인 자동 기록
```

## 5. 퀵 버튼 8종 (워치 스피커 즉시 재생)

| 버튼 | 워치 TTS (즉시) | 폰 Gemma (맥락 반영) |
|---|---|---|
| 🍼 배고프니? | "아가 배고프니? 맘마 줄까?" | 시간·이벤트·페르소나 반영 변형 |
| 👀 일어났니? | "아가 일어났니? 잘 잤어?" | 〃 |
| 🤕 어디 아파? | "왜 그래, 어디 아파?" | 〃 |
| 👍 잘했어 | "아이 잘했어! 우리 아기 최고야!" | 〃 |
| 💖 엄마 여깄어 | "엄마야. 엄마 여깄어." | 〃 |
| 👨 아빠 여깄어 | "아빠야. 아빠 여깄어." | 〃 |
| ❤️ 사랑해 | "사랑해. 엄마가 우리 아기 너무 사랑해." | 〃 |
| ✋ 잠깐만 | "잠깐만, 엄마 바로 갈게." | 〃 |

→ 워치 TTS = 고정 문구 (즉시 재생, 지연 0)
→ 폰 Gemma = 상황·시간·페르소나 반영 동적 문장 (1~3초 뒤 재생)

## 6. 육아 리포트

> 앱이 감지한 이벤트를 자동 누적해서 **일일·주간 리포트**를 생성.

### 6-1. 일일 요약 카드

```
📋 오늘의 하은이 (2026-04-15, 6개월 3일)

울음       🍼 8회  (배고픔 5 · 불편 2 · 원인불명 1)
웃음       😊 12회
기침       🤧 1회
트림       🫧 4회 (수유 후)
방귀       💨 3회
첫 말 후보  💬 1회 ("맘마" — 14:23)
기저귀     🧷 6회 (수동 입력)
수유       🍼 7회 (수동 입력)

⏰ 가장 많이 운 시간대: 02:00~04:00 (3회)
📊 6개월 평균 대비: 울음 +2회, 수유 동일, 수면 확인 필요
```

### 6-2. 개월 수 비교

Gemma 4가 생성하는 **맥락 기반 코멘트**:

```
"하은이는 오늘 8번 울었어요.
 6개월 아기 평균은 하루 약 6회 정도예요.
 새벽 시간대 울음이 잦았는데, 수유 간격이나 실내 온도를 확인해 보시는 것도 좋겠어요.
 첫 말 후보 '맘마'가 감지됐어요! 축하드려요 🎉"
```

### 6-3. 이벤트 종류별 카운터

| 이벤트 | 자동 감지 | 수동 입력 | 카운트 방법 |
|---|---|---|---|
| 울음 | ✅ 마이크/버튼 | — | Gemma care reasoning 결과 |
| 웃음 | ✅ 마이크/버튼 | — | 〃 |
| 기침 | ✅ 마이크/버튼 | — | 〃 |
| 트림 | ✅ 마이크/버튼 | ✅ | 자동+수동 합산 |
| 방귀 | ✅ 마이크/버튼 | — | 〃 |
| 첫 말 후보 | ✅ ASR | — | milestone 로그 |
| 수유 | — | ✅ 워치/폰 버튼 | 수동 기록 |
| 기저귀 | — | ✅ 워치/폰 버튼 | 수동 기록 |
| 수면 시작/종료 | ⚠️ 추론 | ✅ 버튼 | 울음 없는 구간 추론 or 수동 |

### 6-4. 수동 입력 버튼 (워치)

퀵 버튼 아래에 **기록 전용 버튼**:

```
── 기록하기 ──
[🍼 수유 완료]
[🧷 기저귀 교체]
[😴 수면 시작]
[☀️ 수면 종료]
```

탭하면 타임스탬프 자동 기록 → 리포트에 반영.

### 6-5. 주간 트렌드

```
📈 이번 주 하은이 트렌드 (4/9 ~ 4/15)

울음 횟수:  월 10 → 화 8 → 수 7 → 목 9 → 금 6 → 토 8 → 일 8
           전주 대비 -12% ✓
수유 횟수:  하루 평균 6.8회 (6개월 기준 정상 범위)
첫 말 후보: 2회 ("맘마" 4/13, "빠빠" 4/15)
최다 울음 시간: 02:00~04:00 (새벽 수유 패턴)
```

## 7. 워치 설치 (무선 디버깅)

### 7-1. 개발자 모드 활성화 (**워치 본체에서**)

```
워치 설정 → 일반 → 시계 정보 → 소프트웨어 정보
→ "소프트웨어 버전" (L705NKOU3BZB2) 줄을 빠르게 7번 탭
```

⚠️ **폰의 Galaxy Wearable 미러에서는 안 됨. 워치 직접 조작 필수.**

### 7-2. 무선 디버깅

1. 워치: **개발자 옵션 → ADB 디버깅 ON → 무선 디버깅 ON**
2. **페어링 코드로 기기 페어링** 탭 → IP:포트 + 6자리 코드 확인

### 7-3. 연결 (가장 쉬운 방법: 폰 핫스팟)

1. Flip 6: 모바일 핫스팟 ON
2. PC + 워치 모두 해당 핫스팟 Wi-Fi 연결
3. PowerShell:

```powershell
$env:PATH += ";C:\Users\HW-LEE\AppData\Local\Android\Sdk\platform-tools"
adb connect [워치IP]:[포트]
```

### 7-4. 앱 설치

```
Android Studio:
  모듈 → wear
  기기 → SM-L705N (워치)
  Run ▶
```

### 7-5. 연결 유지 팁

- 워치 **개발자 옵션 → Stay awake ON** (충전 중 화면 유지)
- 워치를 **충전기 위에 올려두고 개발**
- 끊기면 `adb connect IP:PORT` 재실행

## 8. 진동 패턴

| 긴급도 | 패턴 | 체감 |
|---|---|---|
| HIGH | 800ms × 3 + 재진동 2회 | 강하게 반복, 못 느낄 수 없음 |
| MEDIUM | 500ms × 3 | 보통 |
| LOW | 300ms × 2 | 가볍게 |

야간 모드에서는 진동 유지하되 **워치 스피커 볼륨만 낮춤** (아기 깨우지 않도록).

## 9. 제스처 모션 캘리브레이션 (선택)

퀵 버튼으로 충분하지만, **손목 모션으로 버튼 없이 발화**하고 싶으면:

1. 제스처 페이지 아래로 스크롤 → "모션 감지 (선택)" 섹션
2. 각 라벨 탭 → 녹화 화면 → **2초간 손목 크게 움직이기** × 3회
3. 8개 다 캘리브 완료 → **▶ 라이브 감지 시작** 탭
4. 손목 모션 → 자동 인식 → 워치 TTS + 폰 전송

### 추천 모션 예시

| 라벨 | 모션 | 특징 |
|---|---|---|
| hungry | 손목을 입 쪽으로 2번 가져감 | 먹는 시늉 |
| awake | 손목을 위로 올림 | 기지개 |
| hurt | 가슴 쪽 2번 터치 | 아픔 표현 |
| well_done | 엄지 척 올리기 | 칭찬 |
| mom_here | 가슴에 손 얹고 유지 | 엄마 |
| dad_here | 가슴에 손 대고 빠르게 두드림 | 아빠 |
| love | 가슴 위 하트 그리기 | 사랑 |
| wait | 손바닥 펴서 앞으로 밀기 | 정지 |

## 10. 문제 해결

| 증상 | 해결 |
|---|---|
| 워치 스피커 소리 안 남 | 워치 설정 → 소리·진동 → 미디어 볼륨 확인 |
| 퀵 버튼 탭해도 폰 TTS 안 나옴 | 폰 SoundCue 앱 실행 중인지 확인 |
| 진동 약함 | 워치 설정 → 진동 강도 최대 |
| 리포트 숫자가 안 맞음 | 수동 입력(수유·기저귀) 빠뜨린 것 확인 |
| 야간 톤 안 바뀜 | 사용자 설정에서 야간 모드 시간대 확인 |
| 캘리브 녹화 실패 | 센서 권한 확인 + 녹화 중 손목 크게 움직이기 |
